两数之和

题目链接

题目

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素不能使用两遍。
示例:
给定 nums = [2, 7, 11, 15], target = 9
因为 nums[0] + nums[1] = 2 + 7 = 9
所以返回 [0, 1]

题意理解

寻找两个数，使之和为target，且输入的数组中只有两个数满足条件。

解法一：暴力解法

两次for循环，寻找满足条件的两个数。时间复杂度O(n2)，空间复杂度O(1)。
代码略。

解法二：二轮哈希表

对于两数之和来说，有这样一个特性，对于固定一个数，有固定的另一个数使两数之和为target。因此，我们可以遍历数组，将数组内容存入哈希表，然后再遍历一次数组，借助哈希表寻找每个数对应的另一个数是否存在。
总的来说，遍历两次数组，时间复杂度O(n),借助了一个哈希表，空间复杂度O(n)。
代码略

解法三：一轮哈希表

解法三是解法二的优化，经过思考可发现，可以边遍历边插入哈希表，同时检查是否存在对应的数。。在进行迭代并将元素插入到表中的同时，我们还会回过头来检查表中是否已经存在当前元素所对应的目标元素。如果它存在，那我们已经找到了对应解，并立即将其返回。

三数之和

题目链接

题目

给你一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？请你找出所有满足条件且不重复的三元组。

注意：答案中不可以包含重复的三元组。

示例：

给定数组 nums = [-1, 0, 1, 2, -1, -4]，

满足要求的三元组集合为：
[
[-1, 0, 1],
[-1, -1, 2]
]

题意理解

寻找三个数，使其满足之和为0，且得到的结果不能为重复的三元组。

解法一：暴力法

进行三次for循环，时间复杂度为O(n2)，同时需要去重，去重方法与解法二一致（因为java int[]的equals方法没有被重写，无法用HashSet来去重），总的时间复杂度为O（n2+a*blogb）,空间复杂度为O(1)。
代码略。

解法二：哈希法

由两数之和的思路可得，进行一次for循环，将问题转化为两数之和的场景，再进行一轮哈希表解决。需要注意的是，与两数之和的场景不同，三数之和结果不唯一且不能重复，因此需要对结果进行去重，比较麻烦，去重方法为对结果进行排序后再比较，时间复杂度为O（mnlogn）,空间复杂度为O(1)。总的时间复杂度为O（n2+ablogb）,空间复杂度为O(n)。
代码略。

解法三：排序+双指针法

排序+双指针法是对暴力法的改进。
基于不重复的理念，在三重循环的框架下，我们需做到以下几点。

  • 二重循环的数不小于一重循环
  • 三重循环的数不小于二重循环
  • 在同一循环中，相邻两次枚举元素不能相同，例如当枚举到（1，2，3）时，三重循环的下一次枚举不应该是3，应该跳过3.

但这样还是属于三重循环的范围，没有利用到排序带来的特性。经过排序，要使a+b+c=0,那么a<b<c,且a<0。如果我们固定两个数a、b，那么就只有唯一的c能满足条件。此时要寻找下一个满足条件的b和c时，b增大，那么c就应该缩小。也就是说可以使用左右两个指针进行遍历，这种遍历是同时的。因此，总的方法时间复杂度O(nlogn+n2),空间复杂度O(1)
