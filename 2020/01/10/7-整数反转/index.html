注意:
假设我们的环境只能存储得下 32 位的有符号整数，则其数值范围为 [−231, 231 − 1]。请根据这个假设，如果反转后整数溢出那么就返回 0。

思路

整数反转和字符串反转的思路相似，不同的是整数可以通过数学方法实现反转，比较简单。

具体思路如下：

  1. 取整数中的一个数字，考虑到取整数的第一个数字的难度远大于最后一个数字，因此从整数的末位开始取。
  2. 将取得的数字添加在之前取得数字组成的整数 rev 末尾。
  3. 重复以上流程，直至原整数无法再取到数字，返回 rev 值。

需要注意的是，在思路的第二步，可能存在数值溢出的情况，因此需要加入溢出检测环节，按题目要求，反转后整数溢出返回0。

其中，为了防止数值溢出现象的产生，不能将第二步取得的结果直接与取值范围比较，而是应该在溢出产生前预知，因此也要注意如何写溢出检测的条件。

以正整数为例

  1. 当 rev > Integer.MAX_VALUE/10 的时候，第二步产生的结果一定溢出。
  2. 当 rev = Integer.MAX_VALUE/10 的时候，第二步产生的结果可能溢出，如果要添加的数字大于7（Integer.MAX_VALUE%10）的时候，
    产生的结果溢出，否则不溢出。

负整数同理

  1. 当 rev < Integer.MIN_VALUE/10 的时候，第二步产生的结果一定溢出。
  2. 当 rev = Integer.MIN_VALUE/10 的时候，第二步产生的结果可能溢出，如果要添加的数字小于-8（Integer.MIN_VALUE%10）的时候，产生的结果溢出，否则不溢出。

代码

class Solution {
    public int reverse(int x) {
        int rev = 0;
        while(x!=0){
            // 取出整数的最后一位数字
            int pop = x % 10;

            // 取出后的整数
            x = x / 10;

            // 溢出检测
            boolean i = rev > Integer.MAX_VALUE/10 ||
                    (rev == Integer.MAX_VALUE/10 && pop > Integer.MAX_VALUE%10);
            boolean j = rev < Integer.MIN_VALUE/10 ||
                    (rev == Integer.MIN_VALUE/10 && pop < Integer.MIN_VALUE%10);

            if(i){
                return 0;
            }
            if(j){
                return 0;
            }

            rev = rev * 10 + pop;

        }
        return rev;
    }
}

结果和复杂度分析

  • 时间复杂度：O(log(x))，x 中大约有 log10x个数字。
  • 空间复杂度：O(1).
