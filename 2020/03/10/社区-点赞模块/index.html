本社区点赞模块主要依赖 Redis 实现缓存机制，MySQL实现点赞信息的持久化。

需求分析

点赞模块主要有以下需求：
1、 在页面展示当前问题或评论的赞同数。
2、 在页面展示当前用户对当前问题或评论是否已赞同。
3、 用户点击点赞按钮后，能实现点赞和取消点赞两种动作。
4、 用户被点赞后，能收到相关信息的通知。

数据库设计

数据库结构包括以下表的设计：
1、 question 表：记录问题的相关信息，在此模块中相关的为点赞数的记录 like_count


2、 comment 表：记录评论的相关信息，在此模块中相关的为点赞数的记录 like_count


3、 favor 表：记录点赞记录的相关信息，其中 id 为点赞记录的 id，是该表的主键，是自动增长的；parent_id 是被点赞主体（问题或评论）的 id；type 标识着被点赞主题是问题还是评论；like_poster_id 是当前点赞用户的id，status 表示当前点赞记录是否有效；gmt_create 和 gmt_modified 点赞记录的创建时间和修改时间。
